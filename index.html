Understanding Video File Sizes & Storage

What is File Size in Video?

Video file size is determined by three main factors: resolution (pixels), frame rate (fps), and bitrate (Mbps). The formula is simple: File Size = Bitrate × Duration. Higher resolution, higher frame rate, or higher bitrate all increase file size. Professional formats like ProRes RAW offer high quality but large file sizes, while compressed formats like H.264 are smaller but lower quality.

ProRes vs Other Codecs

🎬 ProRes RAW

Quality: Uncompressed (12-bit RAW)

Use: Professional color grading, preservation

File Size: Very large (~900 Mbps at 4K 24fps)

Best For: High-end production, archival

🎞️ ProRes 422 HQ

Quality: Compressed (high quality)

Use: Professional editing, broadcast

File Size: Moderate (~500 Mbps at 4K 24fps)

Best For: Post-production, streaming delivery

📹 DNxHD/DNxHR

Quality: Mezzanine codec (Windows alternative)

Use: Professional editing on Windows

File Size: Similar to ProRes

Best For: Windows-based workflows

💻 H.264 / H.265

Quality: Highly compressed

Use: Web streaming, final delivery

File Size: Very small (~5-50 Mbps)

Best For: Distribution, streaming

Bitrate Explained

Bitrate is the amount of data processed per second, measured in Megabits per second (Mbps). Higher bitrate = higher quality but larger file size. For example:

  • 100 Mbps: Professional quality, ProRes RAW-adjacent, highest quality
  • 50 Mbps: High quality, ProRes 422 HQ, suitable for color grading
  • 25 Mbps: Good quality, ProRes 422, common in production
  • 10 Mbps: Acceptable quality, streaming, lower bandwidth
  • 5 Mbps: Compressed, streaming, small file sizes

Storage Planning: The 3-2-1 Rule

Professional video production follows the 3-2-1 backup rule: 3 copies of your data, on 2 different types of media, with 1 copy stored off-site.

  • 3 Copies: Original + 2 backups (minimum)
  • 2 Media Types: Fast SSD for editing, slow HDD for archive
  • 1 Off-site: Cloud storage (AWS, Google Cloud) for disaster recovery

Common Storage Mistakes

❌ Only 1 Copy of Footage

Hard drives fail. Always maintain 3 copies: original, backup, and off-site. ProRes RAW is too valuable to lose.

❌ Underestimating Storage Needs

4K RAW is ~410 GB/hour. Plan for 2-3x the raw footage size (proxies, backups, working files).

❌ Using Consumer Drives for Video

Consumer USB drives/SD cards fail under sustained writes. Use professional SSD/HDD rated for video (e.g., Atomos, G-Technology).

❌ No Proxy Workflow

Editing 4K RAW is slow. Create H.264 proxies for editing, conform to RAW on export. 10x faster.

Frequently Asked Questions

How much storage do I need for ProRes RAW?

ProRes RAW at 4K 24fps generates ~410 GB per hour. For an 8-hour shoot: 3.28 TB. With 3-2-1 backup strategy: ~10 TB total storage (3 copies across different drives/locations).

What's the difference between ProRes RAW and ProRes 422 HQ?

ProRes RAW is uncompressed 12-bit RAW data (maximum quality). ProRes 422 HQ is lossy-compressed 10-bit. ProRes RAW is ~900 Mbps; ProRes 422 HQ is ~500 Mbps. Use RAW for color grading; HQ for editing/delivery.

Can I edit 4K ProRes RAW directly?

Technically yes, but impractical. ProRes RAW is very large and demanding. Standard workflow: (1) Record ProRes RAW, (2) Create H.264 proxies, (3) Edit proxies, (4) Conform to RAW on final export.

How do I calculate bitrate if I don't know it?

Bitrate = (Width × Height × Bit Depth × Frame Rate) / (1,000,000) Mbps. For 4K RAW 12-bit 24fps: (4096 × 2160 × 12 × 24) / 1,000,000 = 3,145 Mbps (raw), compressed ProRes RAW ~920 Mbps.

What's the cheapest reliable storage for video?

For budget: 8TB external HDDs (~$100-150 each). For performance: NVMe SSD drives. For archival: LTO tape (most cost-effective at scale). Buy 3 of the same drive model to maintain redundancy.

Should I use SSD or HDD for video storage?

SSD for active editing (fast access), HDD for archive (cost-effective at scale). Many professionals use: SSD for current project, HDD for backup, cloud for off-site copy.

Does frame rate affect file size?

Yes, directly. 60fps generates 2.5x more data than 24fps (same resolution, codec). A 4K 60fps ProRes RAW clip is huge (~1 TB/hour). 24fps is more manageable.

Can I use cloud storage for ProRes RAW?

For backup/off-site copy: yes (AWS S3, Google Cloud, Dropbox). For active editing: no (too slow). Typical workflow: local SSD for editing, cloud for backup.
